SLIDE 2: EXECUTIVE SUMMARY & MARKET CONTEXT
--------------------------------------------
- Overview: Sea Horizon is a bespoke management solution designed to empower boutique and independent hotels with high-end digital infrastructure.
- Industry Problem: The hospitality sector is rapidly moving away from legacy systems; hotels that don't adapt face decreasing occupancy rates and operational chaos.
- Our Value Proposition: We provide a unified platform that manages the entire guest lifecycle—from discovery to check-out—ensuring a "Luxury-First" digital experience.

SLIDE 3: DETAILED PROBLEM STATEMENT
-----------------------------------
- 1. Operational Fragmentation: Relying on disparate tools (Excel, paper, email) leads to a fragmented view of hotel health.
- 2. Double-Booking Crisis: Without real-time synchronization, rooms are often sold twice across different channels, causing severe brand damage.
- 3. Financial Leakage: Manual invoice generation and revenue tracking lead to "Invisible Losses" due to human error and unaccounted services.
- 4. Security Risks: Storing guest data in unencrypted physical or digital formats poses significant privacy and compliance risks (GDPR/Data Privacy).

SLIDE 4: STRATEGIC PROJECT OBJECTIVES
-------------------------------------
- Automation: To automate 90% of manual data entry tasks, allowing staff to focus on guest hospitality rather than paperwork.
- Real-Time Inventory: Implementing a "Single Source of Truth" database that updates availability across the portal in milliseconds.
- Analytical Insight: Providing administrators with data-driven reports to identify peak seasons and optimize room pricing.
- Responsive Excellence: Ensuring the platform is accessible and stunning on every device, from ultra-wide monitors to smartphones.

SLIDE 5: SYSTEM ARCHITECTURE & DESIGN PATTERNS
-----------------------------------------------
- 3-Tier Architecture: 
    1. Presentation Layer (UI): HTML5, CSS3, and JavaScript for an interactive experience.
    2. Logic Layer (Server): PHP 8.2 handling the core business rules and algorithms.
    3. Data Layer (Storage): MySQL 8.0 for secure, relational data management.
- Design Pattern: Utilizing a modular approach for easier maintenance and future feature scalability.

SLIDE 6: TECHNICAL STACK & PERFORMANCE METRICS
----------------------------------------------
- Backend Engine: PHP 8.2 (High-speed execution and vast library support).
- Database: MySQL (ACID compliant, ensuring transaction reliability).
- Interactivity: JavaScript (DOM manipulation for dynamic room filtering and instant UI updates).
- Security Layer: PHP Data Objects (PDO) for secure database queries; Bcrypt for password encryption.
- Performance Goal: Achieve sub-500ms page load times to minimize guest bounce rates.

SLIDE 7: DATABASE ARCHITECTURE (SCHEMA)
----------------------------------------
- Core Entities: 
    - USERS (Personal data, encrypted credentials, role-based access).
    - ROOMS (Status, price, category_id, unique amenities).
    - BOOKINGS (Check-in/out dates, total_price, status_flag).
    - AMENITIES & CATEGORIES (Metadata for rich room descriptions).
- Data Integrity: Implementing foreign key constraints and cascading deletes to prevent orphaned records.

SLIDE 8: GUEST EXPERIENCE (FRONT-END MODULES)
---------------------------------------------
- Smart Search: An intelligent search engine that calculates availability by checking date-range overlaps in the booking table.
- Interactive Catalog: High-resolution room galleries with dynamic pricing based on room category.
- Personal Dashboard: A private portal for guests to manage upcoming stays, view historical invoices, and update profile security.

SLIDE 10: ADMINISTRATIVE CONTROL CENTER
---------------------------------------
- Inventory Hub: A master dashboard for admins to toggle room availability and update pricing in real-time.
- Reservation Management: A queue-based system for admins to review, approve, or cancel booking requests with automated email notifications.
- Reporting Engine: Visual data summaries showing monthly revenue, occupancy percentages, and guest demographics.

SLIDE 11: SECURITY & COMPLIANCE FRAMEWORK
------------------------------------------
- Input Sanitization: Stripping malicious scripts from user inputs to prevent Cross-Site Scripting (XSS).
- Prepared Statements: Using PDO to neutralize SQL Injection (SQLi) attempts.
- Session Management: Secure cookie handling and session regeneration to prevent hijacking.
- Password Hashing: Industry-standard Bcrypt hashing ensures that even in a breach, user passwords remain unreadable.

SLIDE 12: DEVELOPMENT METHODOLOGY
----------------------------------
- Approach: A hybrid Waterfall-Agile methodology.
- Phase 1: Requirement Gathering & UML Design (ERDs and Use-Case Diagrams).
- Phase 2: Core Backend Development (Database setup and PHP CRUD logic).
- Phase 3: UI/UX Implementation (CSS styling and JS interactivity).
- Phase 4: Rigorous Testing (Unit testing for booking logic; UI testing for responsiveness).

SLIDE 13: KEY TECHNICAL CHALLENGES OVERCOME
--------------------------------------------
- The Availability Algorithm: Writing complex SQL queries that accurately identify free rooms by comparing requested date ranges against existing 'Booked' entries.
- UI Consistency: Using advanced CSS Grid and Flexbox techniques to maintain a premium "Luxury Hotel" feel across all browsers and screen resolutions.

SLIDE 14: FUTURE ENHANCEMENTS (ROADMAP 2027)
